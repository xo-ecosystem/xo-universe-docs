# Activation Plan

This page outlines the current short-term activation strategy for XO.

!!! note
    The activation plan is deliberately staged. XO should grow in public only as fast as it can remain understandable and credible.

## Goal

Move XO from internal architecture toward a readable, living public system.

## Current activation principle

Do not try to expose everything at once.

Instead:

- publish only the clearest public explanations
- activate the most believable public flows first
- show mechanical credibility before broad ambition
- build outward from one or two real examples

## Near-term priorities

### 1. Establish the public documentation layer

This repository is part of that step. It gives XO a clean public reading surface before deeper code and strategy repositories are opened up.

### 2. Clarify the system state

The public should understand what already exists, what is prototype-level, and what is next.

### 3. Activate a flagship flow

A public-facing artifact flow, such as a drop or sealed narrative object, should demonstrate the trust continuum in action.

### 4. Connect story to system

The documentation, visuals, domain constellation, and activated flows should reinforce each other instead of feeling fragmented.

## 30-day direction

### Week 1

- establish the public docs structure
- write the core explanation pages
- define the first public architecture and system-state language

### Week 2

- refine diagrams and supporting visuals
- align the strongest public-facing domains and entry points
- identify the first activation artifact or trust flow

### Week 3

- publish or connect a real artifact flow
- document it clearly as an example of XO in action
- keep the scope tight and credible

### Week 4

- expand from the first successful public flow
- bring in stronger ecosystem links, additional visuals, or supporting modules
- keep unfinished internals private until they earn a public place

## Working rule

XO should grow in public only as fast as it can remain understandable.

!!! tip
    Read this together with [System State](system-state.md) to connect current reality with the next steps, and with [XO Universe Map](xo-universe-map.md) to see where activation flows sit in the larger system.
